Mixture Contour Forecasting

Hannah M. Director, Adrian E. Raftery, and Cecilia M. Bitz

Introduction

This vignette introduces how Mixture Contour Forecasts (MCF) for probabilistic predictions of sea ice presence as in Probabilistic Forecasting of the Arctic Sea Ice Edge with Contour Modeling (Director et al. 2020). MCF predictions are based on a two-component mixture model which weights a contour forecast and a 10-year climatology. The 10-year climatology is here defined for each grid box as the proportion of times in the past ten years that sea ice is present in observations. The contour forecast is more complicated. This vignette assumes the contour forecast has already been computed. For details on the contour forecast method, see Director et al. (2020). For illustration of code to compute the contour forecast, see the IceCast Package vignette Fitting and Generating Contour Forecasts.

In this vignette, we’ll illustrate how the IceCast package can be used to fit an MCF forecast. The focus of the vignette is on how to use the code. For methodological details and performance analysis, see Director et al. (2020). As an example, we’ll make and plot a MCF prediction for September 2008 at a 1.5-month lead time using training data from September 2005-2007.

Finding Weights

To get started, we’ll load the IceCast R package.

library("IceCast")

To compute the mixture weights, the forecasts and observations should be arranged in an array of dimension of the number of training years x longitude x latitude. The contour forecasts, the climatology forecasts, and the observations for the training years for September 2005-2007 are stored in the IceCast package as cont_9_2005_2007, clim_9_2005_2007, and obs_9_2005_2007. Their dimensions are all 3 x 304 x 448.

To find the weights for the mixture model, we input the model outputs and observations into the fit_weights function. We also need to supply a prop_area matrix, which is provided in the IceCast package for the Seas of the Arctic on a Polar Stereographic grid. This is just a matrix of dimension longitude by latitude that sums to 1. Each element gives the proportion of the total area being analyzed that is contained in each grid box.

weight <- fit_weights(mod1 = cont_9_2005_2007, mod2 = clim_9_2005_2007,
                      obs = obs_9_2005_2007, prop_area = prop_area)

Making a Forecast

To issue a MCF forecast for 2008, we use the weights computed in the previous step and matrices of dimension longitude x latitude that give the contour and climatology predictions for the forecast year. For September 2008, these values are stored in the package as cont_9_2008 and clim_9_2008. We compute the MCF forecast as follows

MCF_9_2008 <- wght_mod(w = weight, mod1 = cont_9_2008, mod2 = clim_9_2008)

Plotting the Forecast

However, plot the resulting MCF prediction and the observed ice edge for September 2008.To do this, we’ll load two packages to aid in visualization: the viridis (Garnier et al. 2017) and fields (Nychka et al. 2017) R package.

library("fields")
library("viridis")

We’ll now take a few steps to make the plots more visually appealing. First, we’ll record some information about the data’s North Polar Stereographic Grid.

#dimension of polar Sterographic grid
nX <- 304; nY <- 448 

#x-coordinates
xmn = -3850; xmx = 3750 
xBd <- seq(xmn, xmx, 25)

#y-coordinates
ymn = -5350; ymx = 5850 
yBd <- seq(ymn, ymx, 25)

Since we’re considering a September forecast, most of the Arctic will not have sea ice. So, we’ll zoom in on the area where we do have sea ice.

#zoomed-in area
xBdInd <- 60:215; xBdN <- length(xBdInd) 
yBdInd <- 130:310; yBdN <- length(yBdInd)

Our plots will also be clearer if we designate grid boxes that are land and grid boxes that are outside the analysis region. So, we’ll let land grid boxes have value \(101/100\) and the areas outside the analysis region have value \(102/100\).

#land 
MCF_9_2008[is.na(MCF_9_2008)] <- 101/100 

#region and ocean masks
all_regions_mask <- conv_to_grid(all_regions)
non_reg_ocean <- matrix(nrow = nX, ncol = nY, data = 0)
non_reg_ocean[all_regions_mask == 0] <- 1
MCF_9_2008[non_reg_ocean == 1] <- 102/100

Finally, we’ll plot the MCF prediction of the probability of sea ice presence compared to the observed ice edge.

#plot MCF probabilities
layout(matrix(data = c(rep(1, 10), 2), ncol = 11))
par(oma = rep(2, 4), mar = rep(0, 4))
image(xBd[xBdInd], yBd[yBdInd], MCF_9_2008[xBdInd[1:(xBdN - 1)], yBdInd[1:(yBdN - 1)]],
      xaxt = "n", yaxt = "n", col = c(rep(viridis(10), each = 10), "grey", "beige"),
      zlim = c(0, 102/100), xlab = "", ylab = "", main = "Mixture Contour Forecast",
      cex.main = 1.6)

#plot observed ice edge
plot(obs_9_2008_poly,  add = T, border = "red", lwd = 1)

#add legend
legend("bottom", fill = c("grey", "beige", NA), 
       legend = c("land", "Outside Region", "Observed Contour"),
       lty = c(NA, NA, 1), pch = c(22, 22, NA), ncol = 3, cex = 1,
       col = c("white", "white", "red"), lwd = c(NA, NA, 2),
       bg = 'white', border = c("black", "black", "white"),
       pt.cex = c(3, 3, 3), text.font = 2)

#add color bar
plot.new()
image.plot(MCF_9_2008, zlim = c(0, 1), col = viridis(10), legend.only = TRUE,
           legend.width = 4, legend.shrink = .75)

We see that in this case the observed ice edge contour generally goes through regions of non-zero probability. For more general information on the performance of MCF, see Director et al. (2020)
